🧬 四层记忆架构

借鉴人类认知科学，Agent 的记忆从快到慢、从短到长，形成完整的认知层次。

⚡
L0 · 工作记忆 (Working Memory)
⏱ 纳秒级
载体：LLM 的 Context Window（上下文窗口）
内容：当前推理链 (CoT)、System Prompt、最近 N 轮对话、任务执行状态
特点：容量有限（4K~200K tokens），速度最快，Agent 每一步都直接访问
类比：就像你正在思考时脑子里活跃的那些信息
⬇ 溢出时自动分页 / 摘要压缩 ⬇
📖
L1 · 情景记忆 (Episodic Memory)
⏱ 毫秒级
载体：对话历史数据库 / 时间序列存储
内容：完整对话记录、任务执行日志、用户交互历史（带时间戳）
特点：可搜索、可回溯，保留事件的时间顺序和因果关系
类比："我记得上周二用户问过关于退款的事情"
⬇ 提取 / 去语境化 / 结构化 ⬇
🌐
L2 · 语义记忆 (Semantic Memory)
⏱ 毫秒级
载体：向量数据库 (Milvus/Pinecone) + 知识图谱 (Neo4j/Graphiti)
内容：从对话中提炼的事实、用户画像、领域知识、实体关系
特点：按语义相似度检索，支持跨会话知识复用，容量无限
类比："我知道 Python 是一种编程语言，由 Guido 创建"（不记得何时学到的）
⬇ 固化 / 内化 / 训练 ⬇
🔧
L3 · 程序记忆 (Procedural Memory)
⏱ 永久
载体：模型权重 (微调/RLHF) + Prompt 模板 + Tool 定义
内容：学会的技能、行为模式、工作流、条件反射式响应
特点：最慢更新（需要训练），但执行最快最稳定
类比："我会骑自行车"——不需要回忆怎么骑，身体自动执行

🏗️ 主流记忆框架

当前业界最具代表性的三大 Agent 记忆框架，各有独门绝技。

L
Letta (MemGPT)
LLM as Operating System · UC Berkeley · 2023
核心理念是把 LLM 当作「操作系统」，借鉴 OS 的内存管理机制——CPU 寄存器 → RAM → 硬盘，设计了三层虚拟上下文管理。Agent 自己通过 function call 管理自己的记忆，无需外部编排。
🔀 虚拟上下文管理 🤖 Agent 自编辑记忆 📄 分页机制 (Paging) 🔓 无限上下文窗口 🗄️ 可持久化跨会话

关键创新：Agent 自己决定什么时候从 Recall/Archival 中搜索信息、加载到 Core Memory，什么时候把 Core Memory 中的内容归档。就像你自己决定什么时候回忆、什么时候做笔记。

M
Mem0
Self-improving Memory Layer · 2024
定位为通用的记忆中间件，可以嵌入任何 LLM 应用。自动从对话中提取关键信息、去重合并、按需检索。就像给每个用户装了一个"智能笔记本"。
🔌 即插即用 🧠 自动提取 🔄 增量更新 👤 用户级隔离 🎯 Graph Memory

关键创新：全自动的记忆生命周期管理——你只管对话，Mem0 自动提取事实、合并冲突、版本演进。2025年新增 Graph Memory，支持实体关系图谱。

Z
Zep (Graphiti)
Temporal Knowledge Graph · 2025
最前沿的方案！核心是时序知识图谱 (Temporal Knowledge Graph)——不仅记住事实和关系，还记住这些事实什么时候成立。用户上周喜欢 Java，这周改学 Python？Zep 能追踪这种变化。
⏰ 时序知识图谱 📊 Graphiti 引擎 🏆 DMR SOTA 🔀 自动去重合并 🏢 企业级

关键创新：传统知识图谱只记录 "A → B" 的关系。Zep 记录 "A → B (从1月到3月)"、"A → C (从3月起)"。能回答"用户最近喜欢什么？"这类需要时间推理的问题。

🕸️ 知识图谱记忆

为什么知识图谱是 Agent 记忆的前沿方向？它解决了向量检索无法处理的关系推理问题。

向量检索 vs 知识图谱

📦

向量检索 (RAG)

原理：把文本变成向量，按相似度搜索
✅ 擅长：语义相似的内容匹配
✅ 简单易用，开箱即用
❌ 无法理解实体关系
❌ 无法做跨越多跳的推理
❌ 无法追踪时间变化
类比：在一堆文件中搜索关键词

🕸️

知识图谱 (Graph)

原理：实体 + 关系 + 属性构成网络
✅ 擅长：多跳关系推理
✅ 精确的结构化查询
✅ 可追踪时间维度（时序图谱）
❌ 构建成本较高
❌ 需要 LLM 做实体/关系抽取
类比：一张标注了所有人脉关系的思维导图

🏆 最佳实践：混合架构 (Hybrid)

业界共识是向量检索 + 知识图谱双引擎。
向量负责语义召回（粗筛），图谱负责精确推理（精排）。
Zep 的 Graphiti 就是这种混合架构的典型实现：图谱边同时存向量嵌入，实现语义 + 结构双检索。

📊 架构对比总览

一目了然，各框架在不同维度上的能力对比。

📅 Agent 记忆技术演进

从简单的上下文窗口到时序知识图谱

2023 之前
纯上下文窗口 + 简单对话历史。Agent 没有"记忆"，每次对话几乎从零开始。
2023 Q4
MemGPT 论文发布 — 提出"LLM as OS"理念，引入三层虚拟上下文管理，Agent 首次能自主管理记忆。
2024 Q1-Q2
RAG 爆发期 — 向量数据库 + RAG 成为主流记忆方案。但关系推理能力弱的短板逐渐暴露。
2024 Q2-Q3
Mem0 发布 — 首个即插即用的记忆中间件，自动提取-合并-检索。极大降低了接入成本。
2024 Q4
GraphRAG 兴起 — 微软 GraphRAG 引领知识图谱 + RAG 融合浪潮。业界开始重视结构化记忆。
2024 Q4
MemGPT → Letta — 正式更名 Letta，商业化公司成立。框架持续迭代，增强企业级能力。
2025 Q1
Zep 发布论文 + Graphiti 开源 — 时序知识图谱架构横空出世，在 DMR 和 LongMemEval 基准上刷新纪录。
2025 趋势
混合架构成为主流 — 向量 + 图谱双引擎。Mem0 加入 Graph Memory，各家都在融合图谱能力。记忆系统从"可选"变成 Agent 的"标配"。

🎯 如何选择？

🔬

选 Letta

需要 Agent 精细控制自己的记忆、自主决定读写。长任务执行、需要类 OS 的内存管理。

🚀

选 Mem0

快速集成，最低成本加上记忆能力。个性化助手、用户偏好追踪、简单记忆场景。

🏢

选 Zep

企业级复杂场景，需要关系推理、时间追踪、多跳查询。知识密集型 Agent。
