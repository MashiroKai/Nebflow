# Issue 类型选择

> 创建 issue 时，先选择类型，再按对应模板填写。
> 所有模板共用相同的[元信息格式](./templates/_meta.md)和[术语定义](./templates/_glossary.md)。

| 类型 | 适用场景 | 核心关注 | 状态流转 |
|------|---------|---------|---------|
| **Bug** | 修复缺陷 | 根因分析、复现步骤、回归防护 | `草稿 → 已确认 → 排查中 → 已定位 → 修复中 → 待验证 → 已关闭` |
| **Feature** | 新增功能或行为变更 | 需求澄清、接口设计、兼容性 | `草稿 → 评审中 → 待开发 → 开发中 → 待验证 → 已发布` |
| **Refactor** | 代码/架构重构，无行为变更 | 设计合理性、迁移安全、验证手段 | `草稿 → 评审中 → 待开发 → 开发中 → 待验证 → 已完成` |
| **Quick Fix** | 一行修复、文档笔误、配置调整 | 快速闭环、无副作用确认 | `草稿 → 修复中 → 待验证 → 已关闭` |

模板文件位置：

- `docs/issues/templates/bug.md`
- `docs/issues/templates/feature.md`
- `docs/issues/templates/refactor.md`
- `docs/issues/templates/quickfix.md`

---

## 填写规范

1. **必填项**标记为 `*`，其余按需填写
2. **创建时**填写到"范围"为止；开发过程中补充"详细设计/变更"
3. **关闭/发布时**必须填写"复盘"
4. **变更文件**表格在创建时为"计划"，关闭前更新为"实际"

## 状态变更规则

| 状态变更 | 触发条件 | 必须填写的内容 |
|---------|---------|---------------|
| 草稿 → 已确认/评审中 | 创建完成 | 元信息、一句话描述、范围 |
| 已确认 → 排查中 | 开始排查 | 复现信息（Bug） |
| 排查中 → 已定位 | 找到根因 | 根因分析 |
| 已定位 → 修复中/开发中 | 开始编码 | 修复方案/详细设计 |
| 修复中/开发中 → 待验证 | 代码提交 | 变更文件（实际） |
| 待验证 → 已关闭/已发布/已完成 | 验证通过 | 复盘 |
