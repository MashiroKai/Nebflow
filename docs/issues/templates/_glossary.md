# 术语定义

所有术语的取值必须来自下表，禁止自定义。

## 优先级（Priority）

| 级别 | 定义 |
|------|------|
| **P0-阻塞** | 阻塞发布或阻碍其他关键工作，必须立即处理 |
| **P1-高** | 重要功能/修复，计划在近期版本完成 |
| **P2-中** | 有价值，但无紧迫时间要求 |
| **P3-低** | 锦上添花，资源充裕时处理 |

## 严重程度（Severity）

| 级别 | 定义 |
|------|------|
| **Critical** | 系统不可用、数据丢失或安全漏洞 |
| **High** | 核心功能受损，有明显 workaround |
| **Medium** | 非核心功能问题，或影响范围有限 |
| **Low** | 边缘场景、UI 瑕疵、文档错误 |

## 影响范围（Impact）

| 级别 | 定义 |
|------|------|
| **全部用户** | 所有环境/用户均受影响 |
| **部分用户** | 特定条件触发 |
| **仅开发者** | 仅影响开发/构建流程 |
| **无** | 无实际影响（如文档笔误） |

## 复现频率

| 级别 | 定义 |
|------|------|
| **必现** | 100% 复现 |
| **高频** | >50% 概率复现 |
| **偶发** | <50% 概率复现 |
| **仅报告一次** | 仅单次报告，未复现 |

## 风险等级（Risk Level）

| 级别 | 定义 |
|------|------|
| **Low** | 可回滚，影响面可控 |
| **Medium** | 需监控，可能影响关联模块 |
| **High** | 需灰度，可能引入系统性问题 |

## 文件操作（Action）

| 操作 | 含义 |
|------|------|
| **Create** | 新建文件 |
| **Modify** | 修改现有文件 |
| **Delete** | 删除文件 |
| **Rewrite** | 大面积重写（变更行数 >50%） |
| **Move** | 移动/重命名文件 |

## 标签（Label）

| 标签 | 用途 | 类型前缀 |
|------|------|----------|
| `backend` | 服务端逻辑 | |
| `frontend` | Web UI / JS / CSS | |
| `infra` | 构建、部署、CI/CD | |
| `db` | 数据库、存储相关 | |
| `api` | 接口契约、协议 | |
| `perf` | 性能优化 | |
| `security` | 安全相关 | |
| `docs` | 文档、注释 | |
| `test` | 测试相关 | 必须附加 `test` |
| `deps` | 依赖升级/变更 | |

## 状态流转

不同 issue 类型使用不同的状态机，禁止混用。

### Bug / Performance Regression

```
草稿 → 已确认 → 排查中 → 已定位 → 修复中 → 待验证 → 已关闭
```

| 状态 | 含义 |
|------|------|
| **草稿** | 刚创建，信息待补充 |
| **已确认** | 问题真实存在，可复现或已接受报告 |
| **排查中** | 正在定位根因 |
| **已定位** | 根因已找到，待制定修复方案 |
| **修复中** | 正在实施修复 |
| **待验证** | 修复已完成，等待验证 |
| **已关闭** | 验证通过或关闭原因已确定 |

### Feature

```
草稿 → 评审中 → 待开发 → 开发中 → 待验证 → 已发布
```

| 状态 | 含义 |
|------|------|
| **草稿** | 需求待完善 |
| **评审中** | 方案评审中 |
| **待开发** | 评审通过，等待排期 |
| **开发中** | 正在实现 |
| **待验证** | 实现完成，等待验证 |
| **已发布** | 已上线/交付 |

### Engineering Task

```
草稿 → 评审中 → 待开发 → 开发中 → 待验证 → 已完成
```

| 状态 | 含义 |
|------|------|
| **草稿** | 方案待完善 |
| **评审中** | 方案评审中 |
| **待开发** | 评审通过，等待排期 |
| **开发中** | 正在实施 |
| **待验证** | 实施完成，等待验证 |
| **已完成** | 验证通过 |

### Tech Debt

```
草稿 → 已记录 → 待偿还 → 偿还中 → 已关闭
```

| 状态 | 含义 |
|------|------|
| **草稿** | 债务待描述清楚 |
| **已记录** | 债务已记录，等待触发偿还 |
| **待偿还** | 触发条件已满足，等待排期 |
| **偿还中** | 正在偿还 |
| **已关闭** | 债务已清理 |

### Quick Fix

```
草稿 → 修复中 → 待验证 → 已关闭
```

| 状态 | 含义 |
|------|------|
| **草稿** | 问题待确认 |
| **修复中** | 正在修复 |
| **待验证** | 修复完成，等待验证 |
| **已关闭** | 验证通过 |

### Test

```
草稿 → 评审中 → 待开发 → 开发中 → 待验证 → 已完成
```

| 状态 | 含义 |
|------|------|
| **草稿** | 方案待完善 |
| **评审中** | 测试策略评审中 |
| **待开发** | 评审通过，等待排期 |
| **开发中** | 正在实现 |
| **待验证** | 实施完成，等待验证 |
| **已完成** | 验证通过 |

## 关闭原因

关闭 issue 时必须选择一项。

| 原因 | 说明 | 适用类型 |
|------|------|----------|
| **已完成** | 代码变更已合并，验证通过 | refactor |
| **已修复** | 问题已解决 | bug / perf / quickfix |
| **已发布** | 功能已上线/交付到生产环境 | feature |
| **已偿还** | 债务已清理 | tech-debt |
| **重复** | 与已有 issue 重复 | 所有类型 |
| **无法复现** | 无法重现问题 | bug / perf |
| **不予处理** | 经评估不处理 | 所有类型（需说明理由） |
| **已过时** | 环境/需求已变化 | 所有类型 |
| **已取消** | 计划变更 | 所有类型（需说明理由） |

## 模板类型映射

`_build.py` 使用以下映射校验模板文件。新增模板类型时，在此添加对应行，无需修改脚本。

| 模板文件 | 状态机 | 关闭原因 |
|----------|--------|----------|
| `bug.md` | Bug / Performance Regression | 已修复, 重复, 无法复现, 不予处理, 已过时, 已取消 |
| `perf.md` | Bug / Performance Regression | 已修复, 重复, 无法复现, 不予处理, 已过时, 已取消 |
| `feature.md` | Feature | 已发布, 重复, 不予处理, 已过时, 已取消 |
| `refactor.md` | Engineering Task | 已完成, 重复, 不予处理, 已过时, 已取消 |
| `tech-debt.md` | Tech Debt | 已偿还, 重复, 不予处理, 已过时, 已取消 |
| `quickfix.md` | Quick Fix | 已修复, 重复, 不予处理, 已过时, 已取消 |
| `test.md` | Engineering Task | 已完成, 重复, 不予处理, 已过时, 已取消 |

## Quick Fix 升级规则

满足以下任一条件时，必须关闭当前 quickfix issue，改用对应完整模板：

| 条件 | 升级目标 |
|------|----------|
| 涉及 >=3 个文件 | bug / refactor |
| 需要架构/接口调整 | refactor |
| 涉及行为变更（非纯修复） | bug / feature |
| 预估工时 > 2h | bug / refactor / feature |
| 需要新增测试覆盖 | bug / refactor |
| 需要多阶段部署 | refactor / feature |
